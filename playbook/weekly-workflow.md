# 週次運用サイクル設計例

Claude Code + AWLL Studio で週次業務を回すための運用設計パターンです。

## 基本サイクル

```
金曜(前週末)        月曜(週初め)        火〜金(平日)        金曜(今週末)
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│① 進捗更新  │ → │② 週次計画  │ → │③ 日次記録  │ → │④ 週次集計  │ → ①へ
│ステータスを │   │今週やること │   │作業記録を   │   │実績と計画の │
│最新化      │   │を整理      │   │残す        │   │差分を集計   │
└──────────┘   └──────────┘   └──────────┘   └──────────┘
```

## 各ステップの詳細

### ① 進捗更新（金曜）

今週のタスクの進捗を最新化します。

```
> 今週のタスク一覧を取得して、完了したものにチェックを入れて
```

**API操作**:
- `GET /api/v1/forms/{formId}/answers` でタスク一覧取得
- `PATCH /api/v1/forms/{formId}/answers/{answerId}` でステータス更新

### ② 週次計画（月曜）

今週やるべきことを整理し、タスクを作成します。

```
> 先週の未完了タスクと今週の新規タスクをまとめて、今週の計画を作成して
```

**ポイント**:
- 前週の未完了タスクを自動で引き継ぎ
- 新規タスクの起票
- 優先順位の設定

### ③ 日次記録（火〜金）

毎日の作業を記録します。

```
> タスク#123に今日の作業内容を記録して: フロントエンド実装完了、テスト作成中、3時間
```

**API操作**:
- Nodes API でサブレコード（作業記録）を追加
  ```
  POST /api/answers/{answerId}/nodes
  {
    "parentRowId": "タスクのrowId",
    "fieldCode": "work_logs",
    "data": { "date": "2026-03-31", "hours": 3, "comment": "フロントエンド実装完了" }
  }
  ```

### ④ 週次集計（金曜）

今週の実績を集計し、サマリーを作成します。

```
> 今週の全タスクの工数を集計して、予定vs実績のサマリーを作成して
```

## スキル設計のポイント

### データベース設計

週次運用に必要な最小限のデータベース構造:

```
プロジェクト管理DB
├── プロジェクト名 (TEXT)
├── ステータス (SELECT: active/completed/on_hold)
├── 開始日 (DATE)
├── 終了日 (DATE)
└── タスク (ARRAY) ← サブテーブル
    ├── タスク名 (TEXT)
    ├── 担当者 (USER)
    ├── ステータス (SELECT: not_started/in_progress/done)
    ├── 予定工数 (NUMBER)
    ├── 実績工数 (NUMBER)
    └── 作業記録 (ARRAY) ← サブサブテーブル
        ├── 日付 (DATE)
        ├── 工数 (NUMBER)
        └── コメント (TEXT)
```

### Claude Code スキルの分割

1つの大きなスキルより、小さなスキルを組み合わせる方が保守しやすい:

```
.claude/skills/project/
├── update-tasks.md     # タスクステータス更新
├── create-plan.md      # 週次計画作成
├── log-work.md         # 日次作業記録
└── weekly-summary.md   # 週次集計
```

### 自動化のレベル

| レベル | 内容 | 例 |
|-------|------|-----|
| L1 | 手動実行 | `/update-tasks` で明示的に呼び出し |
| L2 | 半自動 | 毎週金曜にリマインド → ユーザーが実行 |
| L3 | 全自動 | スケジュール実行（AWLL Studioのスケジューラ機能） |

まずは L1 から始めて、運用が安定したら L2 → L3 へ段階的に自動化することを推奨します。

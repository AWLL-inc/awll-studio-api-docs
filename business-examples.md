# 業務シナリオ別プロンプト集

「Claude Code に何をどう頼めば、どんなものができるか」をシナリオ別にまとめています。
プロンプトはそのままコピー&ペーストしてご利用ください。

## このページの使い方

1. ご自身の業務に近いシナリオを選ぶ
2. プロンプトを Claude Code にコピー&ペースト
3. できあがった結果をブラウザでログインして確認
4. 微調整したい点があれば、続けて「ここをこう変えて」と指示

事前に [はじめての方へ](./GETTING_STARTED.md) で Claude Code のセットアップを完了しておいてください。

## 顧客管理

### 顧客マスタを作る

```text
顧客管理用のデータベースを作成し、顧客一覧画面とサイドバーメニューも用意してください。
フィールドは、顧客名・業種・担当者・連絡先メール・電話番号・初回取引日・備考があれば十分です。
```

### 既存の顧客マスタに項目を追加する

```text
顧客マスタに「ステータス」フィールドを追加してください。
値は「見込み」「商談中」「契約中」「離脱」から選ぶ形にしてほしいです。
追加後、顧客一覧画面にもステータス列を表示してください。
```

### 顧客ステータス別の集計画面を作る

```text
顧客マスタのステータス別件数を、円グラフでダッシュボード化してください。
ステータスごとに色を分けて、クリックすると該当顧客の一覧に遷移する形にしてほしいです。
```

## 案件管理

### 案件管理データベースを作る

```text
案件管理のデータベースを作成してください。
必要な項目: 案件名 / 顧客 (顧客マスタとの紐付け) / 担当者 / 受注予定金額 / 受注確度 / ステータス / 受注予定日 / 備考。
受注確度は 10% / 30% / 50% / 80% / 100% の選択肢にしてください。
作成後、案件一覧画面と、ステータス別カンバンボード画面の 2 種類を用意してほしいです。
```

### 売上見込みダッシュボード

```text
案件マスタから「今月・今四半期・今年」の売上見込み（受注予定金額 × 受注確度）を集計するダッシュボード画面を作ってください。
担当者別の見込み一覧も並べて表示してほしいです。
```

## メール自動化

### 案件ステータスが変わったら担当者にメール送信

```text
案件レコードのステータスが「受注」に変わったとき、案件担当者宛てに「○○案件が受注になりました」とメールを送信するスクリプトを設定してください。
件名と本文は案件名・受注金額・顧客名を含めてほしいです。
```

### 期日リマインダー

```text
案件レコードの受注予定日まで残り 3 日になった案件について、毎朝 9 時に担当者宛てにリマインドメールを送信する仕組みを作ってください。
```

## サブレコードを含むデータ管理

### 顧客と接触履歴

```text
顧客マスタに「接触履歴」というサブテーブルを追加してください。
1 顧客に対して複数の接触履歴を持てるようにし、各履歴は接触日・接触方法 (電話/メール/訪問)・対応者・内容メモを記録できる形にしてほしいです。
顧客詳細画面で接触履歴を追加・編集・削除できるようにしてください。
```

### 案件と見積もり明細

```text
案件マスタに「見積もり明細」というサブテーブルを追加してください。
各明細は、商品名・数量・単価・小計 (自動計算) を持つ形にしてほしいです。
案件詳細画面で明細を追加・編集できるようにし、明細小計の合計を案件の「受注予定金額」に自動で反映してください。
```

## 既存のものを少しずつ改善する

### 画面のレイアウトを変える

```text
顧客一覧画面の列順を「顧客名・ステータス・業種・担当者・連絡先メール」に並べ替えてください。
あと、ステータス列の幅を少し広めにしてほしいです。
```

### よく使う検索を保存

```text
顧客一覧画面に「商談中の顧客のみ表示」「自分が担当の顧客のみ表示」のクイックフィルタボタンを追加してください。
```

### 権限を追加

```text
案件マスタを「営業部だけが閲覧・編集できる」設定にしてください。
他部門のユーザーは案件画面にアクセスできないようにしてほしいです。
```

## うまくいかない時の対処

| 困った状況 | 試すこと |
|----------|--------|
| 思ったものができなかった | 「ここをこう変えて」と具体的に追加で指示。Claude Code が修正します |
| 何も起きない / エラーが返る | 「今のエラーを教えて」と聞いて、エラー内容を Claude Code と一緒に確認 |
| 想定外のデータが入った | 「○○のレコードを全部削除して、もう一度作り直して」と指示 |
| 画面がブラウザに反映されない | ブラウザを強制リロード (⌘+Shift+R / Ctrl+F5)。それでもダメなら 1 分ほど待つ (CDN 反映待ち) |
| そもそも Claude Code が動かない | [Claude Code の公式ドキュメント](https://docs.claude.com/claude-code) をご確認ください |

詳しいトラブルシュートは [はじめての方へ](./GETTING_STARTED.md) のトラブルシュート表をご参照ください。

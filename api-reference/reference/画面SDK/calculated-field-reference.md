# 計算フィールド リファレンス

**対象**: AWLL Studioデータベースビルダー利用者
**最終更新**: 2026-05-16

## 概要

計算フィールド（`CALCULATED`型）は、他のフィールド値を参照して自動的に値を算出するフィールドです。
ユーザーが手動で入力する必要がなく、依存フィールドの値が変更されると自動再計算されます。

---

## 計算タイプ

| タイプ | 用途 | 設定項目 |
|--------|------|----------|
| `FORMULA` | 数式計算 | `formula`（数式文字列） |
| `AGGREGATION` | 配列フィールドの集約 | `sourceArrayField`, `sourceChildField`, `function` |

---

## FORMULA（数式計算）

### 基本構文

数式文字列に **フィールドコード**、**数値リテラル**、**演算子**、**関数** を組み合わせて記述します。

```
formula = "フィールドコード 演算子 フィールドコード"
```

### 演算子

| 演算子 | 説明 | 例 |
|--------|------|-----|
| `+` | 加算 | `price + tax` |
| `-` | 減算 | `total - discount` |
| `*` | 乗算 | `quantity * unit_price` |
| `/` | 除算 | `total / count` |
| `()` | 括弧（優先順位制御） | `(price + tax) * quantity` |
| `-`（単項） | 符号反転 | `-discount` |

**演算子の優先順位**: `*` `/` は `+` `-` より先に計算されます（数学の標準ルール）。

### 関数

#### 丸め関数

| 関数 | 説明 | 例 | 結果 |
|------|------|-----|------|
| `FLOOR(x)` | 切り下げ（負の無限大方向） | `FLOOR(3.7)` | `3` |
| `CEIL(x)` | 切り上げ | `CEIL(3.2)` | `4` |
| `CEILING(x)` | `CEIL`のエイリアス | `CEILING(3.2)` | `4` |
| `ROUND(x)` | 四捨五入（整数） | `ROUND(2.5)` | `3` |
| `ROUND(x, n)` | 四捨五入（小数第n位） | `ROUND(2.355, 2)` | `2.36` |

> **注意**: `ROUND`はBigDecimal精度で計算されるため、浮動小数点誤差の影響を受けません。

#### 数値関数

| 関数 | 説明 | 例 | 結果 |
|------|------|-----|------|
| `ABS(x)` | 絶対値 | `ABS(-3.5)` | `3.5` |
| `MIN(a, b)` | 小さい方 | `MIN(price, max_price)` | 小さい値 |
| `MAX(a, b)` | 大きい方 | `MAX(quantity, 1)` | 大きい値 |

#### 集約関数（配列参照）

| 関数 | 説明 | 構文 |
|------|------|------|
| `SUM` | 合計 | `SUM(配列フィールド[*].子フィールド)` |
| `AVG` | 平均 | `AVG(配列フィールド[*].子フィールド)` |
| `COUNT` | 件数（非nullのみ） | `COUNT(配列フィールド[*].子フィールド)` |

```
SUM(order_items[*].line_total)
AVG(scores[*].value)
COUNT(items[*].memo)
```

### 参照可能なフィールドタイプ

数式内で参照できるのは以下のフィールドタイプのみです：

| フィールドタイプ | 参照可否 | 備考 |
|-----------------|---------|------|
| `NUMBER` | ✅ | 数値フィールド |
| `CALCULATED` | ✅ | 他の計算フィールド（依存順自動解決） |
| `TEXT` | ❌ | 参照不可（数値パースされず0扱い） |
| `DATE` | ❌ | 参照不可 |
| `SELECT` | ❌ | 参照不可 |
| `ARRAY` | ❌ | 直接参照不可（集約関数で参照） |

---

## AGGREGATION（配列集約）

配列（ARRAY）フィールド内の子フィールド値を集約して1つの値を算出します。

### 設定項目

| 項目 | 説明 | 例 |
|------|------|-----|
| `sourceArrayField` | 対象の配列フィールドコード | `order_items` |
| `sourceChildField` | 集約対象の子フィールドコード | `line_total` |
| `function` | 集約関数 | `SUM` / `AVG` / `COUNT` / `MIN` / `MAX` |

### 集約関数

| 関数 | 説明 | 例 |
|------|------|-----|
| `SUM` | 全行の合計 | 明細の金額合計 |
| `AVG` | 全行の平均 | 評価スコアの平均 |
| `COUNT` | 非null行数 | 入力済み行数 |
| `MIN` | 最小値 | 最安値 |
| `MAX` | 最大値 | 最高額 |

---

## 表示オプション

| オプション | 説明 | 例 |
|-----------|------|-----|
| `decimalPlaces` | 表示する小数桁数 | `2` → `1234.56` |
| `unit` | 単位文字列 | `円`, `%`, `kg` |

## 実用例

### 例1: 単価 × 数量

```
formula: "quantity * unit_price"
```

| フィールド | 値 |
|-----------|-----|
| `quantity` | 5 |
| `unit_price` | 1000 |
| **計算結果** | **5000** |

### 例2: 消費税計算（切り下げ）

```
formula: "FLOOR(subtotal * 0.1)"
```

| フィールド | 値 |
|-----------|-----|
| `subtotal` | 11000 |
| **計算結果** | **1100** |

### 例3: 割引適用後の合計

```
formula: "(subtotal + tax) * (1 - discount_rate)"
```

| フィールド | 値 |
|-----------|-----|
| `subtotal` | 10000 |
| `tax` | 1000 |
| `discount_rate` | 0.1 |
| **計算結果** | **9900** |

### 例4: 見積書パターン（ARRAY内計算 + 集約 + 税 + 合計）

データベース構造:

```
order_items (ARRAY)
  ├── product_name (TEXT)
  ├── quantity (NUMBER)
  ├── unit_price (NUMBER)
  └── line_total (CALCULATED) → formula: "quantity * unit_price"

subtotal (CALCULATED) → AGGREGATION: SUM(order_items[*].line_total)
tax      (CALCULATED) → formula: "subtotal * 0.1"
total    (CALCULATED) → formula: "subtotal + tax"
```

入力データ:

| 商品名 | 数量 | 単価 | 小計（自動） |
|--------|------|------|-------------|
| ノートPC | 5 | 1000 | 5000 |
| マウス | 3 | 2000 | 6000 |

計算結果:

| フィールド | 計算式 | 結果 |
|-----------|--------|------|
| `subtotal` | SUM(5000, 6000) | 11000 |
| `tax` | 11000 * 0.1 | 1100 |
| `total` | 11000 + 1100 | **12100** |

### 例5: 四捨五入と最大値制限

```
formula: "MIN(ROUND(base_price * 1.08, 0), max_price)"
```

→ 税込価格を整数に四捨五入し、上限価格を超えない値を返す。

### 例6: 配列の平均と件数

```
// 平均スコア
AGGREGATION: AVG(evaluations[*].score)

// 回答済み件数
AGGREGATION: COUNT(evaluations[*].comment)
```

## 計算順序と依存関係

- 計算フィールド同士の依存関係は**トポロジカルソート**で自動解決されます
- 例: `tax`が`subtotal`を参照 → `subtotal`が先に計算される
- **循環依存**（A→B→A）はエラーとなり、計算がスキップされます

### ARRAY内のネスト計算

配列内の計算は**ボトムアップ**で実行されます：

1. 最も深いネストのARRAY内計算を先に実行
2. 上位レベルの集約・計算を順に実行
3. ルートレベルの計算を最後に実行

---

## エラー時の動作

| 状況 | 動作 |
|------|------|
| ゼロ除算（`x / 0`） | `0` を返す |
| 参照フィールドが未入力/null | `0` として計算 |
| 数式の構文エラー | 計算スキップ（レコード保存は継続） |
| 循環依存 | 計算スキップ（警告ログ出力） |
| 未知の関数名 | エラー（サポート関数一覧を表示） |
| `MIN`/`MAX`の引数不足 | エラー（"expects two arguments"） |
| 空の配列に対するAVG | `0` を返す |

---

## 制約事項

- 計算フィールドは**読み取り専用**（手動入力不可）
- ネストの深さは最大**10階層**まで
- 数式内で使用できる文字: 英数字、アンダースコア、演算子、括弧、ドット、アスタリスク、カンマ、空白
- フィールドコードは`[a-zA-Z_][a-zA-Z0-9_]*` の形式
- 関数名は**大文字のみ**（`FLOOR`, `SUM` 等）
